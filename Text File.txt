Autonomous Wildfire Monitoring System -- MVP Design (English)
1. Overview
This MVP defines an autonomous wildfire monitoring system using: - GOES-16/18 for real-time detection (5--10 min) - VIIRS for confirmation and higher spatial resolution - PyTorch deep-learning for hotspot detection & segmentation - LangGraph + CrewAI multi-agent workflows - MapLibre / Leaflet / Cesium for a web-based map viewer - FastAPI backend and PostgreSQL/PostGIS geospatial database

The system is designed to be modular, reproducible, and scalable.

2. Architecture Overview
Data Flow
Ingestion Agent
Downloads GOES imagery every 5--10 min\
Downloads VIIRS imagery when available\
Saves raw files to data/raw/
Preprocessing & Feature Extraction Agent
Converts radiance → brightness temperature\
Performs cloud masking\
Generates patches for ML inference
PyTorch Inference Agent
Runs CNN/Transformer wildfire detector\
Outputs polygons + confidence scores
Fusion & Decision Agent (LangGraph)
Merges GOES + VIIRS detections\
Prioritizes high-risk events\
Sends alerts
Alerting Agent
Telegram API / Email / SMS\
Internal API POST to emergency systems
Map Publishing Agent
Inserts detections into PostGIS\
Exposes via FastAPI → Web Map Viewer
3. Detailed Agent System (MVP)
3.1 Ingestion Agent
Responsibilities - Fetch GOES ABI Band 7, 14 (Fire/Hotspot) - Fetch VIIRS I-Band (375 m) - Organize into datetime folders

Inputs - Satellite data URLs\ - Cron schedule

Outputs - GeoTIFF or NetCDF files

3.2 Preprocessing Agent
Responsibilities - Convert radiance → temperature\ - Resampling\ - Cloud mask\ - Patch extraction

Outputs - Standardized tensors (.pt)

3.3 PyTorch Inference Agent
Model options - ResNet50 wildfire classifier\ - ViT or Swin Transformer for hotspot segmentation\ - YOLOv8-fire (custom training)\ - UNet for fire mask prediction

Outputs - Fire polygons\ - Heatmaps\ - Confidence scores

3.4 Fusion & Decision Agent (LangGraph)
Responsibilities - Combine GOES (fast) + VIIRS (precise)\ - Discard false positives\ - Generate severity score\ - Trigger alerts

3.5 Alerting Agent
Outputs - Alert message\ - Location + bounding box\ - Fire growth indicators

7. Data Schemas (PostGIS)
Table: fire_events
id SERIAL PRIMARY KEY
timestamp TIMESTAMPTZ
confidence FLOAT
source TEXT
geometry GEOMETRY(POLYGON, 4326)
Table: alerts
